# Dating Platform Onboarding Agent

An AI-powered onboarding agent for a dating platform — designed to feel like an effortless mini date rather than a form to fill out. It gathers profile information through natural conversation, supports both text and voice input, and uses semantic memory to thread answers together across the session.

## What It Does

- **Mini-date conversation flow**: LangGraph orchestrates a stateful, multi-turn conversation with contextual acknowledgments generated by the LLM — not canned responses.
- **Semantic memory**: Each answer is stored in a local Chroma vector database using HuggingFace embeddings. Past answers are retrieved by meaning and fed back into the LLM so later responses can reference earlier ones.
- **AI profile extraction**: A local LLM (via Ollama) reads the user's free-form answers and extracts structured profile fields (name, age, location, personality, relationship goals, etc.).
- **Periodic insights**: Every 3 questions, the agent generates a warm, personalized observation about the user based on what they've shared so far.
- **Voice support**: Can speak questions aloud and listen to answers via microphone. Falls back to text if speech recognition fails.
- **Persistent storage**: Full conversation logs and extracted profiles are saved to a SQLite database, linked by session ID.
- **Demo data**: Comes with 9 sample user profiles for testing.

## Stack

| Layer | Technology |
|---|---|
| Conversation flow | LangGraph |
| LLM inference | Ollama (llama2, local) |
| LLM framework | LangChain |
| Vector memory | Chroma + HuggingFace `all-MiniLM-L6-v2` |
| Database | SQLite (SQLAlchemy ORM) |
| Data validation | Pydantic |
| Voice input | SpeechRecognition (Google API) |
| Voice output | pyttsx3 |

## Prerequisites

- **Python 3.10+**
- **Ollama** running locally — install from [ollama.ai](https://ollama.ai), then:
  ```bash
  ollama serve
  ollama pull llama2
  ```
- A microphone (voice mode only)

> **Note on voice quality:** This demo uses `pyttsx3` (offline TTS) and Google's speech recognition API, which are functional but not production-grade. For a real deployment, the recommended upgrades would be:
> - **Text-to-speech**: [ElevenLabs](https://elevenlabs.io) (free tier available, highly expressive voices) or OpenAI TTS (~$15/1M characters, no free tier)
> - **Speech-to-text**: [OpenAI Whisper](https://openai.com/research/whisper) or [Deepgram](https://deepgram.com) for better accuracy, streaming, and longer timeouts

## Setup

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Seed the database with demo profiles (optional)
python seed_db.py

# 3. Run
python run_onboarding.py
```

When prompted, choose `text` or `voice` mode.

## How It Works

1. A new session UUID is created and a `Conversation` record is opened in the DB.
2. LangGraph steps through 11 questions, one node at a time.
3. Each answer is:
   - Saved to the `Message` table
   - Embedded and stored in Chroma (local `./chroma_db/`)
   - Passed to the LLM alongside semantically similar past answers to extract updated profile fields
4. The LLM generates a contextual acknowledgment for each answer, optionally referencing what the user said earlier.
5. Every 3 questions, a personalized insight is generated from the accumulated profile.
6. At the end, the user confirms their profile and it's saved to the `User` table.
